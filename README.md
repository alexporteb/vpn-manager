# 🔒 VPN Manager

**Полноценный VPN-сервер + интерактивный TUI-менеджер для Ubuntu/Debian**

Один скрипт — четыре протокола. Установка и удаление в одну команду.

---

## 🚀 Установка (одна команда)

```bash
curl -fsSL https://raw.githubusercontent.com/alexporteb/vpn-manager/main/setup-vpn.sh -o /tmp/setup-vpn.sh && sudo bash /tmp/setup-vpn.sh
```

## 🗑️ Удаление (одна команда)

```bash
curl -fsSL https://raw.githubusercontent.com/alexporteb/vpn-manager/main/setup-vpn.sh -o /tmp/setup-vpn.sh && sudo bash /tmp/setup-vpn.sh --uninstall
```

> Полностью удаляет все пакеты, конфигурации, сертификаты, пользователей, правила firewall и бинарник vpn-manager.

---

## ✨ Возможности

| Протокол | Шифрование | Подсеть |
|----------|-----------|---------|
| **IKEv2** | ChaCha20-Poly1305 / AES-256-GCM | `10.10.10.0/24` |
| **IKEv1** | AES-256-SHA1 (XAUTH) | `10.10.11.0/24` |
| **L2TP/IPsec** | IPsec + MSCHAP-v2 | `10.10.12.0/24` |
| **PPTP** | MPPE 128-bit | `10.10.13.0/24` |

- 🖥️ **TUI-менеджер** — управление через красивое текстовое меню
- 👥 **Мультипротокольные пользователи** — один пользователь добавляется сразу во все 4 протокола
- 🔐 **Автогенерация PKI** — CA + серверный + клиентские сертификаты (RSA 4096)
- 🔁 **Идемпотентность** — повторный запуск не ломает существующую конфигурацию
- 🌐 **Автоопределение IP** — внешний IP сервера определяется автоматически
- 🧱 **Firewall** — автонастройка UFW + iptables NAT/MASQUERADE

## 📋 Что устанавливается

| Пакет | Назначение |
|-------|-----------|
| `strongswan` | IKEv2 / IKEv1 / IPsec |
| `xl2tpd` | L2TP-туннели |
| `pptpd` | PPTP-сервер |
| `ufw` + `iptables` | Firewall и NAT |

### Открываемые порты

| Порт | Протокол | Назначение |
|------|---------|-----------|
| `500/udp` | UDP | IKE (Internet Key Exchange) |
| `4500/udp` | UDP | IPsec NAT Traversal |
| `1701/udp` | UDP | L2TP |
| `1723/tcp` | TCP | PPTP |
| `22/tcp` | TCP | SSH (сохраняется) |

## 📁 Структура файлов

```
/etc/vpn-manager/
├── users.db          # База пользователей (username:password:psk)
├── server_ip.conf    # IP-адрес сервера
└── psk.conf          # Pre-Shared Key для L2TP/IKEv1

/etc/ipsec.d/
├── cacerts/
│   └── caCert.pem    # Корневой сертификат CA
├── certs/
│   ├── serverCert.pem
│   └── <user>.crt    # Клиентские сертификаты
└── private/
    ├── caKey.pem
    ├── serverKey.pem
    └── <user>Key.pem

/usr/local/bin/vpn-manager  # TUI-менеджер
```

## 🔧 Настройка клиентов

### Windows (IKEv2)

1. **Параметры** → **Сеть и Интернет** → **VPN** → **Добавить VPN**
2. Поставщик: **Windows (встроенный)**
3. Адрес сервера: `ваш IP`
4. Тип VPN: **IKEv2**
5. Тип входа: **Имя пользователя и пароль**
6. Введите логин и пароль из карточки пользователя

### macOS / iOS (IKEv2)

1. **Настройки** → **VPN** → **Добавить конфигурацию VPN**
2. Тип: **IKEv2**
3. Сервер: `ваш IP`
4. Remote ID: `ваш IP`
5. Аутентификация: **Имя пользователя**
6. Введите логин и пароль

### Android (L2TP/IPsec)

1. **Настройки** → **Сеть** → **VPN** → **Добавить VPN**
2. Тип: **L2TP/IPsec PSK**
3. Адрес сервера: `ваш IP`
4. Общий ключ IPsec: PSK из карточки
5. Введите логин и пароль

### Linux (strongSwan)

```bash
sudo apt install network-manager-strongswan
# Далее через NetworkManager → VPN → IKEv2
```

## ⚠️ Важные замечания

- **PPTP** считается устаревшим и небезопасным — используйте IKEv2 когда возможно
- Скрипт рассчитан на **чистую установку** Ubuntu/Debian, но идемпотентен при повторном запуске
- Все пароли хранятся в `/etc/vpn-manager/users.db` (доступ только root, `chmod 600`)
- После добавления/удаления пользователей strongSwan перезагружает конфигурацию автоматически
